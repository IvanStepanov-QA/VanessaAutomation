# ЗАДАНИЕ 3 Проверка доступности реквизитов в зависимости от значения переключателя “Вид товара”.

## 3.1. Первая часть задания

В форме элемента справочника “Товары” есть такой функционал: если переключатель “Вид” устанавливается в позицию “Услуга”, то реквизиты Артикул, Поставщик и Штрихкод становятся недоступными. Если вернуть переключатель в позицию “Товар”, то эти реквизиты снова становится доступными.

Определите, какие случаи необходимо протестировать и автоматизируйте проверку этих случаев.

Для проверки доступности элементов используйте шаги из Библиотеки шагов:

* `И я жду доступности элемента с именем "ИмяЭлемента" в течение 20 секунд`
* `И я жду недоступности элемента с именем "ИмяЭлемента" в течение 20 секунд`

Для определения имени элементов управления формы используйте Исследователь формы.

### Ожидаемый результат 

В качестве результата выполнения задания необходимо создать отдельный запрос на слияние (merge request, MR), содержащий фиче-файл с указанными сценариями и приложить ссылку на этот MR в качестве решения на обучающей платформе https://edu.infostart.ru 


## 3.2. Вторая часть задания 

В логике управления доступностью есть ошибки (не технические). Попытайтесь обнаружить хотя бы одну. Направление поиска обсуждали на вебинаре по итогам первого блока обучения. Посмотрите запись перед выполнением задания, если отсутствовали на вебинаре онлайн.

Зафиксируйте обнаруженный дефект в виде задачи в гитлабе (issue), кратко описав суть проблемы. Создайте к этой задаче запрос на слияние. 

Локально в своей рабочей копии репозитория переключитесь на ветку, созданную для этого запроса на слияние.

Реализуйте в отдельном фиче-файле сценарий, который описывает ожидаемое поведение (соответственно, сценарий должен падать, демонстрируя наличие дефекта/ошибки).

Поместите его в эту ветку (add + commit) и отправьте изменения на сервер (push).

Снимите у запроса на слияние признак "черновик" и назначьте ревьюером тренера курса (@akuntashov).

**Примечание:** Если вы затрудняетесь автоматизировать такой сценарий, используя готовые шаги из встроенной библиотеки VA, то сформулируйте шаги сценария произвольным образом, но с использованием ключевых слов `Дано`/`Когда`/`Тогда`, соблюдая их семантику. Разместите его в этом фиче-файле, но добавьте также в этом фиче-файле тег `@IgnoreOnCIMainBuild`, чтобы исключить этот сценарий из сборочной линии.
